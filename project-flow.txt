Tableli Platform — Complete End-to-End Project Flow (বাংলা ভার্সন)

আমি আপনার সম্পূর্ণ project structure এবং এখন পর্যন্ত implement করা modules বিশ্লেষণ করেছি। নিচে Tableli Platform-এর সম্পূর্ণ End-to-End Flow দেওয়া হলো, যেখানে Service Provider (Chef), Consumer (Customer), এবং Platform Admin — তিনটি side-এর workflow ব্যাখ্যা করা হয়েছে।

1. Service Provider Journey (Chef)
A. Registration & Setup
Authentication (/auth)
Provider প্রথমে registration করবে এবং chef role নির্বাচন করবে।

Features:
- Email verification & OTP verification
- Secure authentication & JWT-based login system

Profile Setup (/profileSetup)
বুকিং রিসিভ করার আগে provider-কে profile setup করতে হবে।
Profile তথ্য:
- Full Name & Display Name, About Me, Location (City, Country)
- Cuisine Specialties & Category (yacht, villa, personal)
- Menu Details (Starting price)
- Visual Portfolio (Dish images, Event highlights)
- Identity Verification

Stripe Onboarding & Escrow Account
বুকিংয়ের টাকা সরাসরি ব্যাংক একাউন্টে পাওয়ার জন্য Chef-কে তার Stripe Account কানেক্ট করতে হবে (/api/v1/payment/stripe-account-onboarding)।

B. Menu & Service Management
Menu Creation (/menuCreate)
Provider তার menu বা dishes create করবে।
Service Details:
- Menu Title, Price per Person, Courses
- Availability Management (Available dates, service windows)

C. Booking & Client Interaction
Incoming Bookings (/booking-chef)
Customer booking করলে provider notification পাবে।
Notification System (/notification)
Provider real-time notification পাবে:
- New Booking Request (booking_created)
- Payment Held in Escrow (payment_hold)
- Payment Released (payment_released)

Booking Approval System
Provider pending bookings দেখতে পারবে এবং:
- Confirm -> Booking Confirmed
- Cancel -> Booking Cancelled

Real-time Chat System (/conversations & Socket.IO)
বুকিং কনফার্ম করার আগে বা পরে Chef সরাসরি Customer-এর সাথে রিয়েল-টাইম চ্যাট করতে পারবে। এখানে Typing Indicator এবং Message Seen/Read Receipt ফিচার আছে।

2. Consumer Journey (Customer)
A. Chef Discovery
Search System (/search)
Customer nearby chef বা specific cuisine search করবে।

B. Booking a Chef
Availability Check & Booking Creation (/booking-chef)
Customer guest count, date, arrival time, এবং location দিয়ে booking request submit করবে। সিস্টেম স্বয়ংক্রিয়ভাবে Total Amount ক্যালকুলেট করে বুকিং জেনারেট করবে।

Secure Payment & Escrow (/payment)
- Customer স্ট্রাইপ চেকআউটের মাধ্যমে পেমেন্ট করবে।
- পেমেন্ট হওয়ার পর টাকাটি সরাসরি Chef-এর কাছে না গিয়ে Platform-এর Escrow-তে 'HOLD' হয়ে থাকবে।

Real-time Chat & Notifications
- Customer তার বুকিংয়ের যেকোনো আপডেট (Chef confirm করেছে কি না) নোটিফিকেশনে পাবে।
- Customer যেকোনো সময় Chef-এর সাথে Socket.io-এর মাধ্যমে লাইভ চ্যাট করতে পারবে।

3. Platform Admin Journey (Super Admin)
Admin Dashboard (/dashboard)
Admin পুরো platform monitor করতে পারবে।

Escrow Release & Payout System (/payment/capture/:paymentId)
- Chef তার সার্ভিস কমপ্লিট করার পর Admin ম্যানুয়ালি টাকা রিলিজ করবে।
- রিলিজ করার সাথে সাথে স্বয়ংক্রিয়ভাবে:
  - 15% Platform Commission এডমিনের একাউন্টে জমা হবে।
  - 85% টাকা সরাসরি Chef-এর Stripe একাউন্টে (Bank-এ) ট্রান্সফার হয়ে যাবে।
  - ডাটাবেজে স্ট্যাটাস 'SUCCESS' হবে এবং বুকিং 'completed' হয়ে যাবে।

Moderation System (/report)
Admin reported content manage করবে এবং শেফদের প্রোফাইল Approve করবে।

4. Technical Infrastructure (Behind the Scenes)
Database Layer (MongoDB + Mongoose)
- Models: User, Profile, Booking, Payment, Notification, Conversation, Message, etc.

Authentication Middleware
- Role-based Access Control (RBAC): Only Chef can accept bookings. Only Users can book. Only Admins can release escrow payments.

Payment Gateway (Stripe)
- Stripe Connect (Destination Charges)
- Stripe Webhooks (checkout.session.completed) পেমেন্ট অটোমেটিক্যালি ট্র্যাক করার জন্য।

Real-time Communication (Socket.IO)
- Private 1-to-1 Messaging
- "Typing...", "Stop Typing" Events
- "Mark as Read" & "Seen" Receipts
- REST API for fetching Chat History (Inbox & Messages)

Overall System Flow:
Customer Search Chef -> Book Chef -> Customer Makes Payment -> Payment is "HOLD" (Escrow) -> Chef Notified & Chat Opens -> Chef Completes Job -> Admin Captures Payment -> 15% Platform Fee / 85% to Chef -> Booking Completed -> Notification Sent.
